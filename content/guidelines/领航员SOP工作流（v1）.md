## 🎯 领航员核心定位

### 角色定义
**成长领航员**是"过程的守护者"和"团队的陪跑教练"，不是传统的"助教"。

### 核心价值
确保"成长过程"的发生，让学员在12周内获得可迁移的素养和能力：
- 团队协作能力
- 反思迭代习惯
- AI高效协同技能

## 🎯 核心职责矩阵

### 1. 技术准备责任人
**目标**: 确保学员技术环境100%就绪

**具体任务**:
- [ ] **环境检查**: 一对一确认每位学员的技术环境
  - VPN配置和可用性
  - AI编程工具（Cursor、GitHub Copilot等）
  - 协作平台（Gather Town、GitHub）
  - 开发环境（Node.js、Python等）
- [ ] **账号与付费**: 确认工具付费问题已解决
- [ ] **家长沟通**: 特殊情况学员的家长沟通
  - 电子设备使用限制
  - 时间投入说明
  - 获取理解和支持

### 2. 团队状态观察者
**目标**: 维护团队"健康度"

**观察维度**:
- [ ] **参与度监控**: 学员在讨论中的参与程度
- [ ] **边缘化预防**: 关注慢热型或内向学员
- [ ] **沟通质量**: 团队成员间沟通是否顺畅
- [ ] **情绪状态**: 学员的学习状态和情绪变化

### 3. 导师与团队协调者
**目标**: 连接项目架构师与学生团队

**协调任务**:
- [ ] **定期同步**: 向导师汇报团队进展、状态和障碍
- [ ] **Office Hour准备**: 引导学生为导师答疑做好准备
- [ ] **问题升级**: 及时将技术难题反馈给导师
- [ ] **资源协调**: 协调导师时间与学生需求

### 4. 成长过程引导者
**目标**: 主持反思复盘会，培养反思习惯

**引导任务**:
- [ ] **周复盘主持**: 每周固定时间主持团队复盘
- [ ] **提问引导**: 通过提问而非直接给答案
- [ ] **失败学习**: 帮助团队从失败中快速学习
- [ ] **持续优化**: 引导团队持续优化行动

### 5. AI原生文化传递者
**目标**: 营造AI原生工作流文化

**传递任务**:
- [ ] **工具推广**: 引导学员使用AI工具
- [ ] **工作流实践**: 在项目全流程中实践AI协作
- [ ] **案例分享**: 分享AI协作的成功案例
- [ ] **问题讨论**: 引导讨论AI的局限性和正确使用
- [ ] **提示词指导**: 教授学员使用标准化AI提示词
- [ ] **协作示范**: 以身作则展示AI协作最佳实践

## 🛠️ 工具箱与心法

### 核心心法
1. **提问，而非包办** (Guide, Don't Do)
   - 当学员遇到困难时，第一反应是提问
   - 引导学员思考："你尝试了什么？""AI给了你什么建议？"
   - 推荐合适工具而非直接代劳

2. **脚手架，不是起重机**
   - 核心能力：耐心、同理心、对人的敏锐觉察
   - 帮助学员"自己爬上去"，而非"吊上去"
   - 成功标准：12周后学员能自主反思和推进项目

3. **传递AI好奇心** (Be the AI Evangelist)
   - 对AI原生工作流充满好奇和兴趣
   - 主动分享AI使用案例和心得
   - 引导讨论："AI在哪些地方帮助了我们？在哪些地方'骗'了我们？"
   - 使用标准化提示词库提升AI协作效率
   - 教授学员AI协作的最佳实践和技巧

## 📅 分阶段工作流程

### 阶段A：启动期（Week 1-2）- 奠定基础

#### A1. 技术就绪检查
**时间**: 项目启动前1周
**责任人**: 领航员
**检查清单**:
- [ ] 一对一联系每位学员
- [ ] 确认VPN、AI工具、协作平台已安装配置
- [ ] 测试所有工具可正常运行
- [ ] 解决付费问题（如Cursor等）
- [ ] 特殊情况学员的家长沟通

#### A2. 破冰与融合
**时间**: 项目启动第1周
**责任人**: 领航员 + 导师
**任务清单**:
- [ ] 组织"二次破冰"活动
- [ ] 确保新老成员快速融合
- [ ] 建立团队协作机制
- [ ] 明确角色分工和职责

### 阶段B：核心共学期（Week 3-10）- 引导与观察

#### B1. 维护"同在感"
**时间**: 每次线上会议
**责任人**: 领航员
**执行要点**:
- [ ] 强制执行"开摄像头"规则
- [ ] 确保线上参与度和专注度
- [ ] 营造团队协作氛围

#### B2. 主持反思复盘会
**时间**: 每周固定时段（20-30分钟）
**责任人**: 领航员
**复盘结构**:
- [ ] 上周做得好的？
- [ ] 遇到的困难？
- [ ] 下周如何改进？
- [ ] AI协作经验分享

#### B3. 周间异步创造协调
**时间**: 周中（周一至周五）
**责任人**: 领航员
**协调任务**:
- [ ] 督促团队组织周中共学时间
- [ ] 关注"边缘"学员状态
- [ ] 引导Building in Public记录

### 阶段C：冲刺期（Week 11-12）- 成果整合

#### C1. 成果整合支持
**时间**: Week 11
**责任人**: 领航员 + 导师
**支持任务**:
- [ ] 协助团队整合MVP
- [ ] 指导故事梳理和展示准备
- [ ] 确保项目按时交付

#### C2. 发布日支持
**时间**: Week 12
**责任人**: 领航员
**支持任务**:
- [ ] 协助产品发布会准备
- [ ] 支持学员展示成果
- [ ] 记录成长轨迹

## 📊 日常管理流程

### 每日站会（15分钟）
**时间**: 每天固定时间
**参与者**: 团队成员 + 领航员
**议程**:
1. 昨日完成情况（5分钟）
2. 今日计划（5分钟）
3. 遇到问题（5分钟）

### 周度管理循环

#### 周一：周计划制定
- [ ] 回顾上周复盘结果
- [ ] 制定本周关键目标
- [ ] 分配具体任务
- [ ] 确认里程碑

#### 周三：中期检查
- [ ] 检查任务完成情况
- [ ] 识别和解决问题
- [ ] 调整计划（如需要）
- [ ] 提供支持和指导

#### 周五：周总结
- [ ] 总结本周成果
- [ ] 识别问题和改进点
- [ ] 准备周末复盘
- [ ] 规划下周重点

#### 周末：共学与复盘
- [ ] 核心共学时间（2小时）
- [ ] 周复盘会（20-30分钟）
- [ ] 周计划会（20分钟）
- [ ] 共学与协作（80分钟）

- [ ] **红色预警**: 连续2天无进展
- [ ] **黄色预警**: 任务延期或质量问题
- [ ] **绿色正常**: 按计划推进

## 🚨 问题处理流程

### 技术问题处理
1. **自助解决**: 引导学员先尝试自己解决
2. **同伴互助**: 组织学员间互相帮助
3. **导师支持**: 升级到导师层面解决
4. **外部资源**: 寻求外部专家支持

### 团队问题处理
1. **沟通问题**: 组织一对一沟通
2. **协作问题**: 调整工作方式和流程
3. **动力问题**: 重新激发学习兴趣
4. **冲突问题**: 及时调解和化解

### 升级机制
- **Level 1**: 领航员直接处理
- **Level 2**: 升级到导师
- **Level 3**: 升级到核心督导组
- **Level 4**: 升级到项目总负责人
